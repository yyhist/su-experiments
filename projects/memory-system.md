# 项目：记忆系统优化

**创建时间**: 2026-05-01
**状态**: 进行中
**优先级**: 高

---

## 目标

降低记忆读取成本，同时防止上下文丢失。当前MEMORY.md索引+topics按需加载+transcripts原始记录的三层模型运转中，但随着项目增多，启动读取链在变长。

## 流程

```
启动 → 读MEMORY.md（轻量索引）→ 按需读topics → 行动中按需读transcripts
```

## 当前进度

| 组件 | 状态 |
|------|------|
| MEMORY.md 轻量索引 | ✅ 运转中 |
| topics/ 按需加载 | ✅ 运转中 |
| transcripts/ 原始记录 | ⚠️ 累积中，未压缩 |
| 决策固化机制 | ⏳ 待设计 |

## 卡点

1. **transcripts压缩** — 无自动压缩，手动grep特定标识符
2. **决策遗忘** — 关键对话决策未固化进文件，依赖context记忆
3. **启动读取链增长** — 项目增多导致MEMORY.md膨胀

## 决策记录

- **2026-05-01** — 一航提出新范式：只保留项目索引，激活时单独阅读。项目放在GitHub，不在本地核心记忆。
